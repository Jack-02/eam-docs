    系统操作¶

    用户操作¶

    用户注册¶

    • [ POST ]    /register

    用户注册新账号。

    示例 
    {
    "identity": "enrique",
    "password": "enrique23"
}

    {
    "code": 0,
    "info": "注册成功"
}

    请求参数¶

    参数 类型 说明
    identity str 用户名
    password str 密码

    响应状态¶

    状态码 说明
    0 注册成功
    -2 请求参数错误
    1 用户名已经存在

    响应数据¶

    无

    用户登录¶

    • [ POST ]    /login

    用户使用用户名和密码登录系统。

    示例 
    {
    "identity": "enrique",
    "password": "enrique23"
}

    {
    "code": 0,
    "info": "登录成功",
    "data": {
        "session": "lox53dvn9k6vma13vkmn4feuvi5vepun",
        "name": "enrique",
        "nickname": "enrique",
        "role_id": 0,
        "entity": null,
        "department": null
    }
}

    请求参数¶

    参数 类型 说明
    identity str 用户名
    password str 密码

    响应状态¶

    状态码 说明
    0 登录成功
    -5 当前用户被冻结
    -2 请求参数错误
    1 用户名不存在
    2 用户名或密码错误

    响应数据¶

    字段 类型 说明
    session int session 信息
    name str 用户名
    nickname str 用户昵称
    role_id int 用户身份
    entity str 实体名称
    department str 部门名称

    用户登出¶

    • [ POST ]    /logout

    用户登出系统。

    示例 
    

    {
    "code": 0,
    "info": "登出成功"
}

    请求参数¶

    无

    响应状态¶

    状态码 说明
    0 登出成功
    1 当前 Session 不可用

    响应数据¶

    无

    飞书帐号绑定或登录¶

    • [ POST ]    /qr_login

    用户绑定飞书帐号或使用飞书帐号进行登录。

    示例 
    {
    "code": "270t16c80d0b4514afd37d85832f20b4",
    "login": 1
}

    {
    "code": 0,
    "info": "get feishu info",
    "data": {
        "session": "lox53dvn9k6vma13vkmn4feuvi5vepun",
        "id": 100471,
        "name": "飞书用户_100471",
        "nickname": "EAM-User",
        "role_id": 4,
        "entity": "清华大学",
        "department": null,
        "feishu": 1,
        "auto": 0
    }
}

    请求参数¶

    参数 类型 说明
    code str 登录流程中获取的授权码
    login int 表示当前调用目的为登录或账号绑定（1表示登录，0表示账号绑定）

    响应状态¶

    状态码 说明
    0 绑定或登录成功
    1 飞书 ID 已绑定其他用户

    响应数据¶

    字段 类型 说明
    id int 用户编号
    name str 用户名
    nickname str 用户昵称
    role_id int 用户身份
    entity str 实体名称
    department str 部门名称
    feishu int 飞书绑定情况（已绑定为 1，否则为 0）

    飞书帐号解绑¶

    • [ GET ]    /unbind

    用户解除飞书帐号绑定。

    示例 
    curl https://example.com/unbind

    请求参数¶

    无

    响应状态¶

    状态码 说明
    0 解绑成功

    响应数据¶

    无

    用户信息¶

    用户信息获取¶

    • [ GET ]    /userinfo

    系统获取当前登录的用户信息。

    示例 
    curl https://example.com/userinfo

    {
    "code": 0,
    "info": "成功获取用户信息",
    "data": {
        "id": 100484,
        "name": "飞书用户_100484",
        "nickname": "EAM-User",
        "role_id": 0,
        "entity": null,
        "department": null,
        "feishu": 1,
        "auto": 0
    }
}

    请求参数¶

    无（需要携带包含 sessionId 字段的 cookie）

    响应状态¶

    状态码 说明
    0 获取成功
    -5 当前用户被锁定
    -4 当前 Session 不可用

    响应数据¶

    字段 类型 说明
    id int 用户编号
    name str 用户名
    nickname str 用户昵称
    role_id int 用户身份
    entity str 实体名称
    department str 部门名称
    feishu int 飞书绑定情况（已绑定为 1，否则为 0）

    用户信息修改¶

    • [ POST ]    /changeinfo

    用户修改昵称和密码。

    示例 
    {
    "nickname": "Enrique"
}

    {
    "code": 0,
    "info": "成功修改用户信息"
}

    请求参数¶

    参数 类型 说明
    nickname str 用户昵称
    password str 密码

    响应状态¶

    状态码 说明
    0 修改成功
    -5 当前用户被锁定
    -4 当前 Session 不可用
    -2 请求参数错误

    响应数据¶

    无
