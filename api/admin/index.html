    用户实体信息修改¶

    • [ POST ]    /admin/users

    系统管理员将实体内用户移出实体，或将未分配用户移入实体。

    示例 
    {
    "user_id": 100484,
    "in_entity": 1
}

    {
    "code": 0,
    "info": "Succeed change user's entity"
}

    请求参数¶

    参数 类型 说明
    user_id int 用户编号
    in_entity int 移动方向（0 表示移出，1 表示移入）

    响应状态¶

    状态码 说明
    0 修改成功
    -2 请求参数错误
    3 未找到实体

    响应数据¶

    无

    用户信息更新¶

    • [ PUT ]    /admin/users

    系统管理员修改实体内用户信息。

    示例 
    {
    "user_id": 100484,
    "name": "飞书用户_100484",
    "nickname": "Enrique",
    "role_id": 4,
    "department": "",
    "frozen": 1
}

    {
    "code": 0,
    "info": "Succeed change user role"
}

    请求参数¶

    参数 类型 说明
    user_id int 用户编号
    name str 用户名
    nickname str 用户昵称
    role_id int 用户类型
    department str 用户所在部门
    password str 用户密码
    frozen int 用户锁定状态

    响应状态¶

    状态码 说明
    0 更新成功
    -2 请求参数错误
    1 用户名已经存在
    2 没有操作权限
    3 用户不存在
    4 实体不存在
    5 部门不存在
    6 部门已有资产管理员

    响应数据¶

    无

    组织或用户查询¶

    • [ POST ]    /admin/filter

    系统管理员查询尚无管理员的部门或未分配的员工。

    示例 
    {
    "method": 1,
    "key": ""
}

    {
    "code": 0,
    "info": "Succeed get orgnames",
    "org_name": [
        "校长办公室",
        "副校长办公室"
    ]
}

    请求参数¶

    参数 类型 说明
    method int 查询类别（1 表示查询组织，2 表示查询员工）
    key str 关键词
    department_id int 部门编号

    响应状态¶

    状态码 说明
    0 查询成功
    -2 请求参数错误
    3 当前系统管理员无对应实体

    响应数据¶

    字段 类型 说明
    user_name array[str] 未分配的员工用户名
    org_name array[str] 尚无管理员的部门名称

    用户资产查询¶

    • [ POST ]    /admin/asset

    系统管理员查询实体内用户的资产列表。

    示例 
    {
    "user_id": 100479,
    "page": 1,
    "pagesize": 10
}

    {   
    "code": 0,
    "info": "Succeed get all assets",
    "assets": [
        {
            "id": 544336,
            "name": "电脑",
            "department": "校长办公室",
            "classname": "电脑"
        }
    ],
    "totalsize": 1
}

    请求参数¶

    参数 类型 说明
    user_id int 用户编号
    page int 资产列表页数
    pagesize int 资产列表页面大小

    响应状态¶

    状态码 说明
    0 查询成功
    -2 请求参数错误
    2 分页错误

    响应数据¶

    字段 类型 说明
    assets array[objects] 资产信息
    totalsize int 资产总数

    {
    "code": 0,

}

    无

    响应状态¶

    状态码 说明
    0 同步成功
    1 当前系统管理员未绑定飞书账号

    响应数据¶

    无

    -2 请求参数格式错误
    1 当前系统管理员未绑定飞书账号

    响应数据¶

    无

    部门管理¶

    部门信息查询¶

    • [ GET ]    /admin/orgs

    系统管理员查询实体内部门列表。

    示例 
    curl https://example.com/admin/orgs

    {
    "code": 0,
    "info": "Get Organization Info Successfully",
    "data": [
        {
            "org_id": 48,
            "name": "校长办公室",
            "entity": "清华大学工字厅",
            "admin": null,
            "children": [
                {
                    "org_id": 49,
                    "name": "副校长办公室",
                    "entity": "清华大学工字厅",
                    "admin": "王希勤",
                    "children": []
                }
            ]
        }
    ],
    "totalsize": 1
}

    请求参数¶

    参数 类型 说明
    org_id int 部门编号
    page int 部门列表页数
    pagesize int 部门列表页面大小

    响应状态¶

    状态码 说明
    0 查询成功
    -2 请求参数错误
    1 部门不存在
    3 系统管理员无实体

    响应数据¶

    字段 类型 说明
    data array[objects] 部门信息
    totalsize int 部门总数

    部门创建¶

    • [ POST ]    /admin/orgs

    系统管理员创建部门。

    示例 
    {
    "name": "新闻中心"
}

    {
    "code": 0,
    "info": "Organization Created Successfully",
    "org_id": 50
}

    请求参数¶

    参数 类型 说明
    name str 部门名称
    parent str 所属部门
    admin_name str 资产管理员用户名

    响应状态¶

    状态码 说明
    0 创建成功
    -2 请求参数错误
    2 无操作权限
    3 系统管理员无实体
    4 所属部门不存在
    5 部门名称重复
    7 系统管理员用户在实体中不存在
    8 系统管理员已有部门

    响应数据¶

    字段 类型 说明
    org_id int 部门编号

    部门信息更新¶

    • [ PUT ]    /admin/orgs

    系统管理员更新实体内组织信息。

    示例 
    {
    "org_id": 50,
    "name": "新闻中心",
    "parent": "校长办公室"
}

    {
    "code": 0,
    "info": "Organization Updated Successfully"
}

    请求参数¶

    参数 类型 说明
    org_id int 部门编号
    name str 部门名称
    parent str 所属部门名称
    admin_name str 资产管理员用户名

    响应状态¶

    状态码 说明
    0 更新成功
    -2 请求参数错误
    1 部门不存在
    2 无操作权限
    3 系统管理员无实体
    4 所属部门不存在
    5 部门名称重复
    6 所属部门不能是当前部门的子部门
    7 系统管理员用户在实体中不存在
    8 系统管理员已有部门

    响应数据¶

    无
