    超级管理员¶

    注意

    不加特殊说明，所有请求都需要携带包含 sessionId 字段的 cookie。

    实体操作¶

    全部实体查询¶

    • [ GET ]    /super_admin/entity

    超级管理员分页查询系统中的全部实体，按创建时间倒序排序。

    示例 
    curl https://example.com/super_admin/entity?page=1&pagesize=10

    {
    "code": 0,
    "info": "成功获取实体信息",
    "entities": [
        {
            "id": 9,
            "entityName": "清华大学工字厅",
            "createdAt": 1684845201.64839,
            "adminName": null,
            "adminId": null
        },
        {
            "id": 6,
            "entityName": "北京大学",
            "createdAt": 1684802919.729449,
            "adminName": "user1",
            "adminId": 50467
        },
        {
            "id": 5,
            "entityName": "清华大学",
            "createdAt": 1684604730.795297,
            "adminName": "cyx",
            "adminId": 2
        }
    ],
    "totalsize": 3
}

    请求参数¶

    参数 类型 说明
    page int 查询页数
    pagesize int 页面大小

    响应状态¶

    状态码 说明
    0 查询成功
    1 分页错误

    响应数据¶

    字段 类型 说明
    entities array[objects] 查询得到的实体
    totalsize int 全部实体的个数

    实体信息更新¶

    • [ PUT ]    /super_admin/entity

    超级管理员更新实体的名称、系统管理员用户名等信息。

    示例 
    {
    "entityID": 9,
    "entityName": "清华大学工字厅",
    "adminName": "user15070"
}

    {
    "code": 0,
    "info": "成功更新实体信息"
}

    请求参数¶

    参数 类型 说明
    entityID int 实体编号
    entityName str 实体名称
    adminName str 系统管理员用户名

    响应状态¶

    状态码 说明
    0 更新成功
    -2 请求参数错误
    2 系统管理员不存在
    3 实体名称已经存在
    5 目标系统管理员已为其他身份

    响应数据¶

    无

    实体创建¶

    • [ POST ]    /super_admin/entity

    超级管理员创建新的实体。

    示例 
    {
    "entityName": "人民大学"
}

    {
    "code": 0,
    "info": "Succeed create entity",
    "data": {
        "entityID": 10,
        "createdAt": 1685005468.054969
    }
}

    请求参数¶

    参数 类型 说明
    entityName str 实体名称
    adminName str 系统管理员用户名

    响应状态¶

    状态码 说明
    0 创建成功
    -2 请求参数错误
    2 系统管理员不存在
    3 实体已经存在
    4 实体名称已经存在
    5 试图设置为系统管理员的用户已有其他身份

    响应数据¶

    字段 类型 说明
    entityID int 实体编号
    createdAt float 实体创建时间

    实体删除¶

    • [ DELETE ]    /super_admin/entity

    超级管理员删除已有实体。

    示例 
    {
    "entityId": 10
}

    {
    "code": 0,
    "info": "Succeed delete an entity"
}

    请求参数¶

    参数 类型 说明
    entityId int 实体编号

    响应状态¶

    状态码 说明
    0 删除成功
    -2 请求参数错误
    1 试图删除的实体不存在

    响应数据¶

    无

    查询可委派的系统管理员¶

    • [ POST ]    /super_admin/filter

    超级管理员查找可委派为系统管理员的用户。

    示例 
    {
    "key": "u"
}

    {
    "code": 0,
    "info": "Succeed get usernames",
    "username": [
        "user15326",
        "user15582",
        "user15838",
        "user16094",
        "user16350",
        "user16606",
        "user16862",
        "user17118",
        "user17374",
        "user17630"
    ]
}

    请求参数¶

    参数 类型 说明
    key str 关键字

    响应状态¶

    状态码 说明
    0 查找成功
    -2 请求参数错误

    响应数据¶

    字段 类型 说明
    username array[str] 可委派的用户名列表
