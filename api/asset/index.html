    资产管理员¶

    注意

    不加特殊说明，所有请求都需要携带包含 sessionId 字段的 cookie。

    资产类型¶

    资产管理员定义资产的层级分类树。具体的资产将必须挂靠在某个层 级分类下。该层级分类为资产的自然分类，即品类。支持每种品类定 义是条目型资产还是数量型资产。

    资产类型信息查询¶

    • [ GET ]    /api/asset_type

    资产管理员查询部门内已定义的资产类型

    示例 
    curl https://example.com/asset_type?page=1&pagesize=10

    {
    "code": 0,
    "info": "Succeed getting types",
    "data": [
        {
            "typeId": 123,
            "typeName": "furniture",
            "fatherId": 10,
        }
        {
            "typeId": 134,
            "typeName": "cloth",
            "fatherId": 20,
        }
    ],
    "totalsize": 2
}

    请求参数¶

    该请求方法不需要参数

    响应状态¶

    状态码 说明
    0 查询成功
    1 查询失败，分页错误
    2 查询失败，无权限
    3 查询失败，未定义实体的管理员
    -4 查询失败，session错误
    -5 查询失败，当前用户被冻结

    响应数据¶

    字段 类型 说明
    data array of [objects] 查询得到的资产类型
    totalsize int 全部一级资产类型的个数
    objects数据 类型 说明
    typeId number 资产类型的id
    typeName str 资产类型名
    category number 资产类别，1为数目型资产2为条目型
    children array of object 子资产类型
    fartherId number 父资产类型的id

    资产类型信息添加¶

    • [ POST ]    /api/asset_type

    资产管理员新定义资产类型

    示例 
    {
    "entityId": 12,
    "data": [
        {
            "typeName": "furniture",
            "father_name": 10,
            "category": 1 //1数量 2条目 (必填)
        }
        {
            "typeName": "cloth",
            "father_name": 20,
            "category": 1 //1数量 2条目 （必填）
        }
    ]
}

    {
    "code": 0,
    "info": "Succeed creating types",
    "id": [1,2,3] //刚刚生成的类型的id
}

    请求参数¶

    字段 类型 说明
    typeName str 类型名称
    father_name str 父类型名称
    category number = ½ 类别，1为数量型资产，2为条目型资产

    响应状态¶

    状态码 说明
    0 添加成功
    1 添加失败，父类型不存在或已存在同名类型
    2 添加失败，无权限
    3 添加失败，未定义实体的管理员
    -4 添加失败，session错误
    -5 添加失败，当前用户被冻结

    响应数据¶

    字段 类型 说明
    ids list 添加后该类型的id

    资产类型信息修改¶

    {
    "typeId"： 1，
    "typeName": "furniture",
    "father": 1,
}

    {
    "code": 0,
    "info": "Succeed updating a type"
}

    请求参数¶

    字段 类型 说明
    typeId number 类型id
    typeName str 类型名称
    father_name str 父类型名称

    响应状态¶

    状态码 说明
    0 修改成功
    1 修改失败，类型不存在
    2 修改失败，无权限
    3 修改失败，未定义实体的管理员
    4 修改失败，父类型不存在
    5 修改失败，已存在同名类型
    -4 修改失败，session错误
    -5 修改失败，当前用户被冻结

    响应数据¶

    无

    资产录入¶

    录入操作¶

    • [ PUT ]    /api/asset/add

    资产管理员一次录入多个资产，包括条目型资产和数量型资产

    示例 
    {
    [
        {//资产1
            "name": "table",//资产名称
            "fatherId": 12,//主从关系
            "ownerId": 123,//资产管理员id
            "value": 123,//单价
            "num": 1234,//数量
            "description": "hahaha!hahaha!"//富文本
            "img": "https://...",
            "type": "aaa"
        },
        {//资产2
            "name": "chair",//资产名称
            "fatherId": 123,//主从关系
            "ownerId": 123,//资产管理员id
            "value": 123,//单价
            "num": 1234,//数量
            "description": "hahaha!hahaha!"//富文本
            "img": "https://...",
            "type": "aaa"
        }
    ]
}

    {
    code: 0, 
    info: "Succeed creating task", 
    task_id: 71
}

    请求参数¶

    字段 类型 说明
    asset list 录入的资产信息
