    系统管理模型¶

    系统日志模型 SysLog¶

    系统日志模型 (SysLog) 用于记录本系统的组织架构层面的操作日志。

    字段名 类型 描述 属性
    id BigAutoField 系统日志的唯一标识 主键
    name CharField 系统日志的名称，最大长度为128个字符，默认值为'default_syslog' -
    logtype IntegerField 系统日志的类型，表示包括登入登出在内的不同的操作类型，共12种 -
    time FloatField 系统日志的创建时间戳 -
    content TextField 系统日志的记录内容 -
    entity ForeignKey(Entity) 系统日志所属的实体，关联到实体模型 (Entity)，级联删除 外键关联
    handler ForeignKey(User) 执行操作的用户，关联到用户模型 (User) 外键关联
    receiver ForeignKey(User) 目标用户，关联到用户模型 (User) 外键关联
    department ForeignKey(User) 目标部门，关联到部门模型 (Department) 外键关联
    operation IntegerField 操作的结果，1 表示成功，0 表示失败 -

    资产日志模型 AssetLog¶

    资产日志模型 (AssetLog) 用于记录本系统的资产相关的操作日志。

    字段名 类型 描述 属性
    id BigAutoField 资产日志的唯一标识 主键
    name CharField 资产日志的名称，最大长度为128个字符,默认值为'default_log' -
    logtype IntegerField 资产日志的类型，表示包括资产录入在内的不同的操作类型，共8种 -
    time FloatField 资产日志的创建时间戳 -
    content TextField 资产日志的记录内容 -
    asset ForeignKey(Asset) 资产对象，关联到资产模型 (Asset) 外键关联
    applicant ForeignKey(User) 申请操作的用户，关联到用户模型 (User) 外键关联
    handler ForeignKey(User) 执行操作的用户，关联到用户模型 (User) 外键关联
    receiver ForeignKey(User) 目标用户，关联到用户模型 (User) 外键关联
    operation IntegerField 操作的结果，1 表示成功，0 表示失败 -

    待办任务模型 TodoList¶

    待办任务模型 (TodoList) 用于存储待办任务的信息。一旦任务完成，将会从该模型中删除。

    字段名 类型 描述 属性
    id BigAutoField 待办任务的唯一标识 主键
    name CharField 待办任务的名称，最大长度为128个字符,默认值为'default_todo' -
    todotype IntegerField 待办任务的类别，表示包括资产领用的不同的任务类型，共4种 -
    time FloatField 申请时间的时间戳 -
    asset_id IntegerField 涉及的资产的ID -
    applicant ForeignKey(User) 申请人，关联到用户模型 (User) 外键关联
    handler ForeignKey(User) 处理人，关联到用户模型 (User) 外键关联
    receiver ForeignKey(User) 目标接收者，关联到用户模型 (User) 外键关联
    instance_id CharField 飞书实例的ID，最大长度为128个字符 默认值为空字符串

    异步任务模型 Task¶

    异步任务模型 (Task) 用于存储系统待执行、正在执行、已执行的全部异步任务(主要为资产批量导入)的信息。

    字段名 类型 描述 属性
    id BigAutoField 任务的唯一标识 主键
    name CharField 任务的名称，最大长度为128个字符,默认值为'default_task'
    content TextField 任务需要执行的内容 -
    tasktype IntegerField 任务的类型，表示不同的任务类型 -
    status IntegerField 任务的状态，表示任务的处理状态，0 表示待处理，1 表示处理中，2 表示已完成，3 表示已取消，默认值为0 -
    create_time FloatField 任务的创建时间的时间戳 -
    update_time FloatField 任务的更新时间的时间戳 -
    handle_time FloatField 任务的处理时间的时间戳 -
    creator ForeignKey(User) 任务的创建人，关联到用户模型 (User) 外键关联
    entity ForeignKey(Entity) 任务所在的实体，关联到实体模型 (Entity),级联删除 外键关联
    result IntegerField 任务的执行结果，表示任务的执行结果，0 表示未执行，1 表示执行成功，2 表示执行失败，默认值为0 -
    message TextField 任务的处理信息，记录了成功或错误的详细信息 -
    asset_num IntegerField 任务总共涉及的资产数量 -
    process IntegerField 任务当前的执行进度百分比 -

    告警策略模型 AlertStrategy¶

    告警策略模型 (AlertStrategy) 用于存储用户定义的告警策略的信息。

    字段名 类型 描述 属性
    id BigAutoField 告警策略的唯一标识 主键
    asset_type ForeignKey(AssetClass) 告警策略的监测资产类型，关联到资产类型模型 (AssetClass)，级联删除 外键关联
    target IntegerField 监测对象，表示是监测净值还是数量 -
    relation IntegerField 相对大小关系，表示是小于还是大于 -
    threshold FloatField 阈值，用于判断是否触发告警 -
    department ForeignKey(Department) 从属部门，关联到部门模型 (Department)，级联删除 外键关联
    create_time FloatField 告警策略的创建时间的时间戳 -

    告警消息模型 AlertMessage¶

    告警消息模型 (AlertMessage) 用于存储因满足告警策略条件而触发的告警消息的信息。

    字段名 类型 描述 属性
    id BigAutoField 告警消息的唯一标识 主键
    content TextField 告警消息的内容 -
    strategy ForeignKey(AlertStrategy) 告警消息的对应策略，关联到告警策略模型 (AlertStrategy)，级联删除 外键关联
    time FloatField 告警消息的创建时间戳 -
    department ForeignKey(Department) 告警消息的从属部门，关联到部门表 (Department)，级联删除 外键关联

    应用模型 Application¶

    应用模型 (Application) 用于存储用户自定义的应用信息。

    字段名 类型 描述 属性
    id BigAutoField 应用的唯一标识 主键
    apptype CharField 应用的类型 -
    name CharField 应用的名称 -
    description TextField 应用的说明 -
    entity ForeignKey(Entity) 应用所属的实体，关联到实体表 (Entity)，级联删除 外键关联
    url CharField 应用的url -
    create_time FloatField 应用的创建时间的时间戳 -
    update_time FloatField 应用的更新时间的时间戳 -

    其中，联合唯一(name, entity)，确保应用在同一实体中的名称是唯一的。
