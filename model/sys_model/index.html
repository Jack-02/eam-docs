    组织架构模型¶

    用户模型 User¶

    用户模型(User)用于存储用户的相关信息。

    字段名 类型 描述 属性
    id BigAutoField 用户的唯一标识 主键
    name CharField 用户的名称，最大长度为50个字符，必须是唯一的 索引
    nickname CharField 用户的昵称，最大长度为50个字符，如果未指定则默认为'EAM-User' -
    password CharField 用户密码的散列，最大长度为256个字符 -
    entity IntegerField 用户所在的业务实体的ID -
    department IntegerField 用户所在的部门（最近部门）的ID -
    identity IntegerField 用户身份，取值为1、2、3、4分别代表超级管理员、系统管理员、资产管理员和员工，0表示刚刚注册，还没有身份 -
    created_time FloatField 用户注册的时间戳 -
    frozen IntegerField 用户是否被冻结，0表示未冻结，1表示冻结 -
    feishu_id CharField 飞书的用户ID -
    auto IntegerField 只针对系统管理员，0表示登录时不自动同步飞书用户，1表示登录时自动同步飞书用户 -

    实体模型 Entity¶

    实体模型(Entity)用于存储实体的相关信息。

    字段名 类型 描述 属性
    id BigAutoField 实体的唯一标识 主键
    name CharField 实体的名称，最大长度为128个字符 -
    admin ForeignKey(User) 实体的系统管理员，关联到用户表(User)，当管理员被删除时，设置为NULL 外键关联
    created_time FloatField 实体的创建时间戳 -

    部门模型 Department¶

    部门模型(Department)用于存储部门的相关信息。

    字段名 类型 描述 属性
    id BigAutoField 部门的唯一标识 主键
    parent ForeignKey('self') 上一级部门的外键，关联到自身，级联删除 外键关联
    entity ForeignKey(Entity) 部门的从属实体，关联到实体表(Entity)，级联删除 外键关联
    name CharField 部门的名称，最大长度为128个字符 -
    admin ForeignKey(User) 部门的资产管理员，关联到用户表(User)，当管理员被删除时，设置为NULL 外键关联
    staff TextField 部门的员工列表，以列表样式存储 -
    created_time FloatField 部门的创建时间戳 -
    total_value TextField 部门的总净值，以列表样式存储，每个元素为元组，元组的第一个元素为时间戳，第二个元素为部门总净值，第三个元素为部门名 -
    all_value TextField 包含子部门的总净值，以列表样式存储，每个元素为元组，元组的第一个元素为时间戳，第二个元素为总净值，第三个元素为部门名 -
