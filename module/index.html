    模块设计¶

    模块概览¶

    本项目设计中，包含如下6个模块：

    • UI 模块，位于前端，提供前端展示、与后端通信功能
    • 后端基础模块，位于后端，提供路由、会话管理、辅助功能、用户鉴权功能
    • 组织架构模块，位于后端，提供实体管理、部门管理、用户管理功能
    • 资产架构模块，位于后端，提供资产相关定义、资产管理功能
    • 系统管理模块，位于后端，从系统层面提供日志查看、任务管理、告警设置、应用管理等功能
    • MySQL 数据库模块，即数据库，包含所有表以及表项，提供所有信息的存储功能

    模块设计细节¶

    UI 模块¶

    UI 模块可以分层解耦合为：

    • 网络层
    • 逻辑层
    • UI 层

    网络层¶

    网络层负责与后端进行通信，并将返回的结果更新到 react.useState 提供的 hook 内。

    逻辑层¶

    逻辑层负责当前页面的全局变量管理，在渲染 UI 组件前，将必要的数据更新好，管理数据流向。

    UI 层¶

    UI 层负责面向用户的组件渲染呈现，并与用户的行为产生交互，将更新的信息传递到网络层向后端发送。

    后端基础模块¶

    后端基础模块可以解耦合为：

    • 路由模块
    • 会话管理模块
    • 鉴权模块
    • 辅助功能模块

    路由模块¶

    路由模块处理所收到的 HTTPS 请求的路径，从而确定请求所需要访问的模块，将该请求及其信息转交给相应的模块进行细节处理。

    会话管理模块¶

    会话管理模块根据前端浏览器的请求以及用户权限进行会话 id 分配，并设置会话的过期时间，将所分配的会话 id 以 Cookie 的形式返还给前端。实现对用户权限的管控。

    鉴权模块¶

    提供函数形式的鉴权服务功能。

    辅助模块¶

    提供返回函数，错误包装函数，计时函数等辅助功能。

    组织架构模块¶

    组织架构模块负责实体管理、部门管理、用户管理等功能。该模块可以分为以下几个子模块：

    • 实体管理子模块
    • 部门管理子模块
    • 用户管理子模块

    实体管理子模块¶

    实体管理子模块负责对组织架构中的实体进行管理，包括实体的增删改查等操作。该子模块可以进一步细分为以下几个模块：

    • 实体数据模块：负责实体数据的存储和管理，包括实体的基本信息、属性信息、关系信息等。
    • 实体查询模块：负责实体的查询和检索，支持多种查询方式，如按名称、按属性、按关系等查询。
    • 实体编辑模块：负责实体的编辑和保存，支持实体属性的修改、关系的建立和删除等操作。

    部门管理子模块¶

    部门管理子模块负责对部门进行管理，包括部门的增删改查等操作。该子模块可以进一步细分为以下几个模块：

    • 部门数据模块：负责部门数据的存储和管理，包括部门的基本信息、属性信息、关系信息等。
    • 部门查询模块：负责部门的查询和检索，支持多种查询方式，如按名称、按属性、按关系等查询。
    • 部门编辑模块：负责部门的编辑和保存，支持部门属性的修改、关系的建立和删除等操作。

    用户管理子模块¶

    用户管理子模块负责对用户进行管理，包括用户的增删改查等操作。该子模块可以进一步细分为以下几个模块：

    • 用户数据模块：负责用户数据的存储和管理，包括用户的基本信息、属性信息、关系信息等。
    • 用户查询模块：负责用户的查询和检索，支持多种查询方式，如按名称、按属性、按关系等查询。
    • 用户编辑模块：负责用户的编辑和保存，支持用户属性的修改、关系的建立和删除等操作。

    资产架构模块¶

    资产架构模块负责资产相关定义、资产管理等功能。该模块可以分为以下几个子模块：

    • 资产定义子模块
    • 资产管理子模块

    资产定义子模块¶

    资产定义子模块负责对资产进行定义，包括资产的类型、属性、标签等。该子模块可以进一步细分为以下几个模块：

    • 资产类型模块：负责资产类型的定义和管理，包括资产类型的名称、属性、关系等。
    • 资产属性模块：负责资产属性的定义和管理，包括资产自定义属性的名称、类型、默认值等。
    • 资产标签模块：负责资产标签的定义和管理，包括资产标签的名称、内容等。

    资产管理子模块¶

    资产管理子模块负责对资产进行管理，包括资产的增删改查等操作。该子模块可以进一步细分为以下几个模块：

    • 资产数据模块：负责资产数据的存储和管理，包括资产的基本信息、属性信息、关系信息等。
    • 资产查询模块：负责资产的查询和检索，支持多种查询方式，如按名称、按属性、按关系等查询。
    • 资产编辑模块：负责资产的编辑和保存，支持资产属性的修改、关系的建立和删除等操作。

    系统管理模块¶

    系统管理模块负责从系统层面提供日志查看、任务管理、告警设置、应用管理等功能。该模块可以分为以下几个子模块：

    日志管理子模块¶

    日志管理子模块负责对系统日志\资产日志进行管理，包括日志的查看、搜索等操作。该子模块可以进一步细分为以下几个模块：

    • 日志存储模块：负责系统日志的存储和管理，包括日志的类型、级别、内容等。
    • 日志查询模块：负责系统日志的查询和检索，支持多种查询方式，如按类型、按级别、按内容等查询。

    任务管理子模块¶

    任务管理子模块负责对系统任务进行管理，包括任务的创建、执行、监控等操作。该子模块可以进一步细分为以下几个模块：

    • 任务定义模块：负责任务的定义和管理，包括任务的名称、内容、参数等。
    • 任务执行模块：负责任务的执行和监控，支持任务的并发执行、任务状态的监控等。

    告警管理子模块¶

    告警管理子模块负责对系统告警进行管理，包括告警的设置、触发、处理等操作。该子模块可以进一步细分为以下几个模块：

    • 告警设置模块：负责告警的设置和管理，包括告警的类型、对象、条件等。
    • 告警触发模块：负责告警的触发和处理，支持告警的自动触发、告警状态的监控等。

    应用管理子模块¶

    应用管理子模块负责对系统应用进行管理，包括应用的配置、修改、删除等操作。

    MySql 数据库模块¶

    MySql 数据库模块提供后端所有模型的存储服务，完成各类数据的持久化存储。
